## 1. 마스터 컴포넌트 이동 작업

### 배경

페이지별로 분산된 마스터 컴포넌트를 `[Master] Comp` 페이지로 중앙화하고,
원래 위치에는 인스턴스를 남겨 레이아웃을 유지하는 작업.

### 결과물

- **이동 대상 페이지**: 📍 26.04 4월 학기 + 5월 대치동 (310:12)
- **이동 대상**: 마스터 컴포넌트 7개
- **이동 목적지**: [Master] Comp 페이지 (3294:78637)
- **섹션 ID**: 3303:11351 (이름: `📍 26.04 4월 학기 + 5월 대치동`)

### 발견한 문제 & 해결

**문제**: `figma.moveNodesToPage()` API가 존재하지 않음
```
TypeError: figma.moveNodesToPage: no such property
```

**해결**: `masterCompPage.appendChild(node)` 루프로 대체

```javascript
for (const node of nodesToMove) {
  const parent = node.parent;
  const currentIndex = Array.from(parent.children).indexOf(node);
  
  // 인스턴스 생성 (원래 위치 유지)
  let instance;
  if (node.type === 'COMPONENT_SET') {
    instance = node.defaultVariant.createInstance();
  } else {
    instance = node.createInstance();
  }
  parent.insertChild(currentIndex, instance);
  instance.x = node.x;
  instance.y = node.y;
  
  // 마스터를 [Master] Comp 페이지로 이동
  masterCompPage.appendChild(node);
}
```

**주의사항**: 인스턴스 삽입 후 인덱스가 shift되므로, 각 노드마다 `indexOf`를 fresh하게 재계산해야 함.
