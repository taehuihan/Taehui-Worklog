## 트러블슈팅

**문제 1: 통째복제 → 레이아웃 상수 불일치**
- 원인: cross-page clone은 소스 치수를 그대로 가져옴 → 가이드 상수(밴드폭, 행간격)와 무관
- 해결: 항상 가이드 기반 재구성. 복제는 화면 instance만 가져올 때만 사용

**문제 2: 행 폭 침범 (D4→D5 밴드 침범)**
- 원인: 한 행에 화면을 5개 나열 → 총 폭 1955px > 밴드폭 770px
- 해결: `shift = 총폭 − 밴드폭 = +1185` 로 D5 이후 전체 우측 이동. "행 폭 한계 규칙" 가이드 추가

**문제 3: 트리거 magnet 추정 오판**
- 원인: Figma magnet(AUTO/BOTTOM 등) 데이터로 커넥터 출발 방향 추정 → 타겟 방향 가장자리로 빠짐
- 해결: 커넥터 bbox 시작점 직접 탐색 → position 기반 최근접 컴포넌트 특정. magnet 추정 영구 금지

**문제 4: 손좌표 하드코딩 반복 실수 → connect() 빌더**
- 원인: 매번 좌표를 계산해 하드코딩 → 5가지 패턴 실수 반복 (관통, 팬아웃 미분리 등)
- 해결: `connect(출발, 트리거질의, 도착)` API 빌더 제작 → 자동 탐색·충돌 검사·팬아웃 분리

**문제 5: DEPTH_GAP 500 → 800**
- 원인: 500px에서 down-lane과 화면 간격이 좁아 시각적으로 답답
- 해결: 800으로 재정립, 기존 500 전면 폐기

**문제 6: 서브레이블 모델 오해 (무료/유료 → 행 헤더로 혼동)**
- 원인: "무료/유료 유저 행 분리" 결정 → 서브레이블을 유저 세그먼트 헤더로 오해
- 해결: 기준섹션 16037 직접 실측 → 서브레이블 = 뎁스(열)별 콘텐츠명, 무료/유료 = 상태레이블

**문제 7: 가이드 버그 = 재현 버그 (§2-2 세그먼트 행 나열 규칙)**
- 원인: 06-26에 추가한 §2-2 규칙("세그먼트 행 안 가로 나열·§5 우선")이 잘못됨 → 기억 리셋해도 가이드를 따라 동일 오류 재현
- 해결: §2-2 교정 — 세그먼트는 라벨, 배치는 항상 §5(다른 진입점 = 다른 행). `verifyRowBranching` segmentRowYs 예외 제거

## 아쉬운 점 · 새 리스크 · 한계

- **트리거 자동화 한계**: position 기반 최근접 컴포넌트 탐색도 1~2px 차이로 오선택 가능 → 매 섹션 디자이너 확인 필요
- **가이드 버그 리스크**: 가이드 자체에 버그가 있으면 기억 리셋 후에도 동일 오류 재현 → 가이드 변경 시 항상 샘플 섹션으로 즉시 검증 필요
- **섹션 cascade 재배치**: 화면 추가 시 이후 섹션들 x 이동 필요 — 현재 수동, 자동화 미완
- **cross-page Figma 작업 속도**: get_design_context 반복 호출이 느림 (대형 파일에서 체감)
- **세션 기억 단절**: Claude Code 세션이 바뀌면 가이드를 다시 Read해야 함 → flow-guard hook으로 강제하지 않으면 즉흥 실수 재발

**재사용 프롬프트 — 섹션 신규 생성**

**재사용 프롬프트 — 섹션 업데이트**
```
[소스 링크] 이 링크의 [섹션명]을 업데이트해줘.

섹션 사이즈 변경 시 이후 섹션 cascade 재배치.
```

**체크리스트 — 빌드 전**
- [ ] guide.md §1~9 직접 Read (기억 복원 ❌)

- [ ] 소스 화면+커넥터 전수 실측 → 트리거 맵 작성
- [ ] 다른 트리거→다른 화면 N개 = 분기행 N개 (다른 화면을 한 행에 묶지 말 것)

**체크리스트 — 빌드 후 9단계 수치검증**

## 결론 / 의의 / 한 줄 평

Master 플로우 섹션 생성·업데이트 전 과정을 AI와 협업해 자동화했으며, 핵심 산출물은 `connect-builder.js`(커넥터 자동 연결)와 guide.md §10(업데이트 워크플로)이다.

완전 무인 자동화는 어렵지만(트리거·행 배치 의미 검증은 디자이너 필수), 대량 배치·수치 계산·스크립트 반복 실행은 AI가 맡고 디자이너는 판단에만 집중하는 구조가 만들어졌다.

> 가이드가 SSOT다. 규칙이 틀리면 재현도 틀린다 — 이게 이 실험의 가장 중요한 교훈.
