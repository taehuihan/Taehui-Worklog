# Master 컴포넌트 정리 (-COMP) 가이드

> **master-flow 짝 문서.** 플로우 빌드 후, 그 플로우가 쓰는 **로컬(같은 파일) 마스터 컴포넌트**를 `[플로우 섹션명]-COMP` 섹션으로 모아 관리한다.
> 규칙 SSOT 짝: [`guide.md`](guide.md)(플로우 레이아웃·커넥터) · [`connect-builder.js`](connect-builder.js) · **이 문서**(컴포넌트 정리).
>
> ⚠️ **작성 중 (WIP, 2026-07-09~)** — 규칙을 점진적으로 채우는 중. `/master-flow-new` 스킬 연결·정본화는 태휘 지시 시.

---

## 1) 스타일 (섹션 외형)

- **bg 색 · 마진(360px)**: master-flow [`guide.md` §4·§4-1](guide.md)과 **동일** — 별도 값 정의 없이 그 규칙을 그대로 따른다. (bg fill `#BFBFBF` 기본 · RA 편차 `#DCDCDC` · 좌·상 마진 360)
- **섹션 width**: **바로 위 플로우 섹션의 width와 동일** — 그보다 **더 길어질 수 없다.** 콘텐츠(컴포넌트)가 폭을 넘으면 **다음 행으로 wrap**.
- **위치**: 마스터 컴포넌트가 속한(= 그 컴포넌트를 쓰는) **플로우 섹션의 하단(바로 아래)**.
- **섹션 간 여백**: **500px** — 플로우 섹션 bottom + 500 = -COMP 섹션 top. (master-flow §4-1 "직전 섹션 +500"과 일관)

---

## 2) 대상 선정

- **대상 = 소스 페이지 안에 있는 마스터 컴포넌트만.** 여기서 **소스 페이지** = 그 플로우의 소스 섹션(Master로 클론한 원본)이 있는 페이지. (예: 학습커스텀 → `📍 추천학습 커스텀`)
- → 플로우와 함께 만들어진 **고유 마스터**만 포함. **공유·중앙 컴포넌트 페이지**(`common comp`·`today's recommend comp` 등, 소스 페이지 밖)의 마스터는 **제외**.

---

## 3) 이동 · 배치 · 라벨

- **이동**: 소스 페이지의 대상 마스터 컴포넌트를 **-COMP 섹션으로 이동(relocate).** 인스턴스는 마스터를 따라가므로 안 깨짐.
- **배치 (flow step처럼 tier 열 구조)** — 정해진 width 안에서:
  - **크기 기준 = "쪼개지는 단위"(포함 계층 tier)** — 픽셀 크기 아님. 합성이 부품을 포함하면 상위 tier.
  - **큰(합성) → 작은(부품) 방향으로 좌→우 열(column) 배치.** 최상위 합성이 **맨 좌측**(flow step 1처럼), tier가 깊어질수록 우측.
  - **같은 tier(같은 쪼개지는 단위)의 컴포넌트는 열로 나누지 말고 같은 열에 행(row)으로 쌓는다.** (flow step의 같은 depth 여러 화면이 행으로 쌓이듯. 예: custom-learning card & Frame 2147236912 = 홈/커스텀 직접 부품 = 같은 tier → 세로로 행 배치)
  - **행 바꿈**: 섹션 width(= 위 플로우 섹션 width)를 벗어나면 tier 열 그룹 단위로 다음 행 그룹.
- **이름 라벨**: **표기하지 않는다.**

---

## 4) 검증

**① 수치 검증** (코드 assert):
- **섹션 width == 위 플로우 섹션 width** (동일, 초과 0).
- **섹션 위치**: top == 플로우 섹션 bottom + 500, 좌변 x == 플로우 섹션 x (정렬 동일).
- **마진 360**: 콘텐츠(컴포넌트)는 섹션 좌·상 경계에서 360 이상 안쪽에서 시작, 우측도 width 안.
- **겹침 0**: 컴포넌트끼리 · 다른 섹션과 bbox 충돌 0.
- **누락 0**: 소스 페이지 대상 마스터 수 == -COMP 내 마스터 수.

**② 의미 검증** (수치가 못 잡음):
- -COMP 내 마스터가 **전부 소스 페이지 소속** (공유·타 페이지 마스터 혼입 0) — §2 준수.
- 이동 후 **플로우 화면 인스턴스 정상** — 마스터 따라감, detach·깨짐 0. **원본 소스 섹션 부작용 0**(마스터 빠진 것 외 다른 요소·인스턴스 정상).

**③ 시각 검증** (스크린샷 대조 — ⚠️ 수치가 못 잡는 것):
- **배치 순서**: 큰 요소=우측 / 그 안에 포함된 작은 요소=좌측 (§3) 맞는지.
- **행 바꿈**: 큰 요소 단위로 wrap, 섹션 width 안 벗어남.
- **이름 라벨 없음**(§3) · bg 색·마진 육안 확인.

> ⚠️ **수치검증 ≠ 시각/의미검증.** 배치·겹침·라벨·이동 무결성은 수치만으론 못 잡음 → 스크린샷(`node.screenshot()` 우회 포함) 시각 확인 필수. (master-flow 가이드 §9·§작업순서가드와 동일 원칙)
