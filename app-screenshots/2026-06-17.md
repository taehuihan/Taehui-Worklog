## 1. 목업 폰 화면 이미지 교체 (원근 baked)

마케팅 목업 `google_1`(11620:57575)의 폰 2개 화면을 신규 이미지(이미지1=팔로잉 홈, 이미지2=Investment Insight)로 교체.

**핵심 발견:** 화면 레이어(`👈`, VECTOR + scaleMode FIT)의 fill이 **원근 왜곡이 baked된 PNG**였음. 평면 스크린샷을 그냥 넣으면 각도 안 맞음.

**해결 절차:**
1. 기존 fill PNG에서 화면 quad 코너 검출 (opencv `approxPolyDP`, 좌/우 폰 동일 각도)
2. 신규 평면 스크린샷을 동일 quad로 `cv2.getPerspectiveTransform` + `warpPerspective` (1000:846 종횡비 유지, 2배 출력)
3. `upload_assets(nodeId, scaleMode FIT)`로 fill 교체

**컴포넌트 상속:** `👈`가 목업 컴포넌트 내부 노드라, fill 교체 시 모든 인스턴스(google_1/_2, 앱스토어-6.5형_1/_2, 5.5형_1/_2)에 자동 상속됨 → 별도 작업 불필요.

## 2. google_3·5 → 앱스토어 적용 (라이브 인스턴스)

앱스토어-6.5형/5.5형의 `_3`, `_5`가 구버전 화면 컴포넌트를 쓰고 있어 google_3(vai-home)·google_5(vai-home/chat)로 교체. (`_4`는 이미 동일 컴포넌트라 변경 없음 — 사용자 확인)

**함정:** 단순 `swapComponent`는 오버라이드(커스텀 콘텐츠)가 빠지고 컴포넌트 기본값만 나옴 → google 인스턴스를 `clone()`해서 오버라이드까지 복제해야 함.

**크기 규칙:** 6.5형 = google와 동일 w·h, 5.5형 = ×0.871 비례 축소. x/y는 앱스토어 템플릿값.

## 3. 앱스토어 5.5형 우측 잘림 해결

5.5형은 인스턴스를 270폭으로 줄여야 하는데, vai-home/chat이 반응형이 아니라 resize 시 콘텐츠가 클리핑됨.

**막힌 방법들:**
- `resize`: 고정폭 콘텐츠 미스케일 → 우측 잘림 ❌
- `rescale`: 이 환경에 **`Pretendard` 폰트가 없어** 텍스트 재배치 실패(w:1 붕괴) ❌
- 자식 constraint→SCALE: 인스턴스 자식이 컴포넌트에 잠겨 변경 불가 ❌

**해결:**
- **5.5형_3** (vai-home, 내부 반응형): 라이브 인스턴스 resize로 정상 축소 → 편집 가능 유지
- **5.5형_5** (vai-home/chat, 고정폭): google_5를 PNG export(scale4) → 사각형 image fill(FILL)로 배치. 폰트 무관하게 정상 렌더. (`upload_assets` fill 자동적용 실패 시 imageHash로 `node.fills` 직접 설정)
